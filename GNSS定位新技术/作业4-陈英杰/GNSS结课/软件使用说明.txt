GPSmap 软件使用说明
====================

一、软件定位
GPSmap 是一款基于 Android 平台开发的导航定位 App。软件以高德地图为底图，集成地图显示、实时定位、地点搜索、路线规划、电子围栏、轨迹记录、拍照定位，以及任务二扩展的 GNSS 质量评估、坐标转换、精度评价、误差源分析、异常检测和 GNSS+惯性传感器组合导航展示等功能。

二、运行环境
1. 开发工具：Android Studio。
2. 运行平台：Android 真机优先，建议 Android 7.0 及以上。
3. 网络环境：地图加载、POI 搜索和路线规划需要联网。
4. 定位环境：GNSS 卫星数、C/N0 和 DOP 等质量指标需要真机并在室外开阔环境测试，模拟器通常不能提供真实卫星状态。

三、工程使用方法
1. 解压 GPSmap 工程压缩包。
2. 使用 Android Studio 打开工程根目录。
3. 将本次提供的四个注释版文件替换到对应位置：
   - MainActivity.java → app/src/main/java/com/example/gpsmap/MainActivity.java
   - activity_main.xml → app/src/main/res/layout/activity_main.xml
   - AndroidManifest.xml → app/src/main/AndroidManifest.xml
   - build.gradle → app/build.gradle
4. 点击 Android Studio 顶部提示中的 Sync Now，或手动执行 File → Sync Project with Gradle Files。
5. 执行 Build → Rebuild Project，确认工程没有编译错误。
6. 连接 Android 真机，开启 USB 调试，点击 Run 安装运行。

四、首次运行授权
1. 首次进入 App 时，需要允许定位权限。
2. 使用拍照定位功能时，需要允许相机权限。
3. 建议在系统定位设置中开启“精确位置”。
4. 如果手机开启了开发者选项中的“模拟位置应用”，建议关闭，否则可能影响 GNSS 异常检测判断。

五、基础功能使用
1. 地图定位：进入 App 后，地图会自动定位到当前位置；点击右侧圆形定位按钮可回到当前位置。
2. 地点搜索：在顶部搜索框输入地点名称，点击“搜索”，从搜索结果中选择目的地。
3. 路线规划：选择目的地后，可点击“步行”或“驾车”生成路线；点击“清路”清除路线。
4. 电子围栏：点击底部“围栏”，长按地图设置围栏中心，可通过半径滑动条调整范围；点击“清围”清除围栏。
5. 拍照定位：点击“拍照”，系统调用相机，拍照后在地图上生成拍照定位点；点击拍照点可查看照片和位置信息。
6. 轨迹记录：点击“轨迹”开始记录当前位置移动轨迹，再次点击停止；点击“清轨”清除轨迹。

六、任务二扩展功能使用
1. 默认界面只显示三个摘要卡片：GNSS 质量、定位精度、异常检测。
2. 点击“功能模块”右侧的“展开”，可显示六个功能标签。
3. 点击“卫星质量”，查看可见卫星数、参与定位卫星数、C/N0、星座统计和 DOP 估算。
4. 点击“精度评价”，查看水平精度、速度、航向、定位方式和可用性建议。
5. 点击“坐标转换”，查看 GCJ-02 坐标对应的 WGS84 和 BD-09 坐标。
6. 点击“误差源”，查看当前可能存在的遮挡、多路径、信号弱、几何构型差等误差来源。
7. 点击“异常检测”，查看位置跳变、速度异常、卫星状态矛盾等初步检测结果。
8. 点击“组合导航”，查看手机惯性/磁传感器航向与 GNSS 航向的一致性分析。

七、GNSS 质量真实性测试建议
1. 尽量在室外开阔区域测试，例如操场、广场或楼顶。
2. 对比室外开阔区、楼宇遮挡区、室内窗边三种环境下的卫星数、C/N0、DOP 和精度变化。
3. 可安装第三方 GNSS 测试软件进行对比，若卫星数和 C/N0 变化趋势基本一致，说明本 App 读取的 GNSS 质量数据具有真实性。
4. 模拟器只能验证界面流程，不能作为 GNSS 质量评估结果依据。

八、常见问题
1. 地图不显示：检查网络连接、高德 API Key、包名和 SHA1 配置。
2. 搜索或路线失败：检查网络连接和高德服务权限是否正常。
3. 卫星数一直为 0：确认使用真机、开启精确定位权限、到室外开阔处测试。
4. 拍照功能无法使用：检查相机权限和设备是否有相机应用。
5. Gradle 同步失败：检查 compileSdk 是否已安装，高德 SDK 依赖是否能联网下载。
