四、输入数据准备
----------------------------------------------------------------
不同脚本需要的输入不同，下面分开说。

（一）batch_plot_vtec.py 所需输入
这个脚本需要：
1. 一个输入目录 input_dir；
2. 目录中放置若干 JSON 文件；
3. 文件名至少满足“被点号分隔后长度 >= 4”；
4. 其中：
   - 最后一段后缀必须是 json
   - 倒数第二段必须是 vtec

也就是说，文件名需要类似这种结构：
    xxxx.method.vtec.json
或者：
    hksl.somealgo.vtec.json

如果文件名不满足这个规则，脚本会直接跳过，不会处理。

JSON 内容要求（根据代码行为推断）：
- 整个 JSON 应该是一个字典/对象；
- 键是卫星 PRN，例如：
      "G01", "G02", "C12", "R05"
- 值是一个按历元排列的 TEC/VTEC 数值数组，例如：
      [12.3, 12.5, 13.1, 0.0, 14.2, ...]

示例结构：
{
  "G01": [12.1, 12.3, 12.8, 13.0],
  "G02": [10.2, 10.6, 10.8, 11.1]
}

注意：
- 脚本会把绝对值小于 1.0e-6 的数视作无效值，并替换成 NaN；
- 这意味着你可以把无观测时刻写成 0，但最终图中会显示为断点。

（二）区域电离层模型脚本所需输入

1. RINEX 观测文件；
2. 接收机位置；
3. 卫星位置或至少能从外部补充卫星位置；
4. 双频观测（L1/L2，或 L1/P2 组合）；
5. 输出路径。

特别注意：
在你之前报告里展示的示例代码中，sat_pos 和 rec_pos 在 main() 中被写成了占位值：
    sat_pos = np.array([0, 0, 0])
    rec_pos = np.array([0, 0, 0])

这说明：
- 如果源码还是这个状态，程序虽然能运行到某些步骤，但算出来的电离层延迟没有实际物理意义；
- 运行前必须检查源码里是否已经把这两处替换成真正的卫星位置和接收机位置；
- 如果没有替换，你需要先补全这部分逻辑再跑。

（三）结果目录
建议你提前准备一个专门的输出目录，例如：

五、batch_plot_vtec.py 的详细运行方法
----------------------------------------------------------------
这是目前最容易直接跑起来的脚本，也是功能最明确的脚本。

（一）脚本功能确认
该脚本的主要流程是：
1. 读取输入目录中的所有文件；
2. 挑出符合“*.vtec.json”模式的文件；
3. 对每个文件执行 plot_vtec()；
4. 逐卫星绘制 VTEC 散点图；
5. 保存为 HTML。

（二）命令行运行格式
脚本的调用方式是：

    python batch_plot_vtec.py <input_dir> <year> <doy>

参数含义：
- input_dir：放置 VTEC JSON 文件的目录
- year：年份，例如 2025
- doy：年积日，例如 111

示例：
    python batch_plot_vtec.py G:\我的云端硬盘\代码\vtec_json 2025 111

（三）建议的目录组织方式
建议这样组织：

G:\我的云端硬盘\代码\vtec_json\
    hksl.method.vtec.json
    bjfs.method.vtec.json
    btng.method.vtec.json

运行后会在 input_dir 下自动生成：
G:\我的云端硬盘\代码\vtec_json\plot\

其中保存的 HTML 文件名大概率为：
    原文件名小写.method_name.vtec.html

（四）图形内容说明
生成的图中：
- 横轴：Epoch（历元编号）
- 纵轴：VTEC[TECU]
- 每条不同颜色的散点系列代表一颗卫星
- 0 值会被显示为断点
- 图中带有图例滚动条和工具栏

（五）运行成功的判断方法
如果运行正常，你通常会看到：
1. 没有 Usage 报错；
2. 输入目录下出现 plot 子目录；
3. plot 子目录下出现若干 .html 文件；
4. 双击 HTML 文件可以在浏览器中打开交互图。

（六）最常见报错与处理
1. 报错：Usage: python batch_plot_vtec.py <input_dir> <year> <doy>
   原因：命令行参数不够
   处理：必须同时传入 3 个参数

2. 没有生成任何图
   原因：
   - 输入目录为空
   - 文件名不满足 *.vtec.json 规则
   - JSON 内容格式不对
   处理：
   - 检查文件名
   - 检查 JSON 顶层是不是字典
   - 检查值是不是数值数组

3. ModuleNotFoundError: No module named 'pyecharts'
   原因：没有安装依赖
   处理：
       pip install pyecharts

4. HTML 生成了但图不显示
   原因：
   - JSON 数据全是 0
   - PRN 键对应数组长度不一致
   - 数据结构异常
   处理：
   - 随机检查一个 JSON 文件内容
   - 保证每个 PRN 对应的是数值列表

七、模型程序涉及的核心计算逻辑（便于理解运行结果）
----------------------------------------------------------------
这一部分不是“如何点击运行”，而是帮助你判断程序跑得对不对。

（一）多项式模型
从已读代码看，模型核心是：

    I = a0 + a1*lat + a2*lon + a3*lat^2 + a4*lon^2 + a5*lat*lon

这表示程序会用位置（纬度、经度）拟合电离层延迟的空间分布。

运行正确时，你应当能看到：
- 一组 6 个或更多参数
- 模型有 R²、RMSE、Bias 等评价结果
- 可视化图中真实值与预测值有一定一致性

（二）质量控制
程序会基于均值和标准差做筛选，类似：
- 保留 |x-mean| <= 3*std 的点
- 只保留物理合理范围内的数据

如果你输入数据质量很差，可能出现：
- 样本大量被剔除
- 最终剩余样本太少
- curve_fit 无法收敛

（三）电离层延迟构造
程序会利用双频观测构造延迟，所以你输入的观测文件中必须具有：
- L1
- L2 或 P2

如果只有单频观测，程序大概率无法正常拟合。

（四）RINEX 解析
从可读代码看，程序会读取：
- RINEX 版本和类型
- 站点近似坐标
- 接收机信息
- 天线信息
- 观测类型
- 首末历元时间
- 观测间隔

因此你输入的 RINEX 文件头必须完整，且格式不能乱。

九、常见问题排查清单
----------------------------------------------------------------
1. 程序一运行就退出
   检查：
   - 有没有 main() 入口
   - 是不是脚本只定义了函数但没有调用

2. 报找不到输入文件
   检查：
   - Windows 路径反斜杠是否写对
   - 是否需要原始字符串 r"路径"

   - 卫星位置 sat_pos 是否仍是 [0,0,0]
   - 接收机位置 rec_pos 是否仍是 [0,0,0]
   - 观测值是否正确读入
   - L1/L2 波段是否存在

6. 结果图生成但很奇怪
   检查：
   - 单位是否混了（米、TECU、相位周等）
   - 异常值是否过多
   - 观测历元是否错位
   - 经纬度是否经过标准化后又被错误解释
